# 智能网页搜索与分析系统需求文档

## 1. 项目概述

### 1.1 项目背景
随着互联网信息的爆炸式增长，用户需要高效地从海量网页中获取有价值的信息。本项目旨在构建一个智能Agent，通过自然语言处理和网页搜索技术，自动为用户筛选和分析相关信息。

### 1.2 项目目标
开发一个基于LangChain和LangGraph框架的智能Agent，实现自动化网页搜索、内容提取和信息分析功能。

## 2. 功能需求

### 2.1 核心功能
1. **用户交互**：接收用户的自然语言描述作为搜索和分析需求
2. **百度搜索集成**：根据用户描述自动调用百度搜索引擎获取相关网页
3. **网页内容获取**：自动获取搜索结果中前20个网页的内容
4. **AI分析**：使用本地OLLAMA模型qwen3.5:4b逐项分析每个网页内容
5. **结果整理**：综合分析结果，输出整理后的结论

### 2.2 详细功能描述

#### 2.2.1 用户需求理解
- 接收用户的自然语言查询
- 解析用户意图和所需信息类型
- 生成适合百度搜索的关键词

#### 2.2.2 网页搜索
- 调用百度搜索API或模拟搜索
- 获取搜索结果列表（标题、URL、摘要）
- 记录搜索结果的相关性评分

#### 2.2.3 网页内容提取
- 访问搜索结果中的每个网页
- 提取网页正文内容
- 处理不同网页结构（HTML解析、内容清理）
- 限制内容大小以提高处理效率

#### 2.2.4 AI智能分析
- 使用LangChain调用本地OLLAMA qwen3.5:4b模型
- 针对每个网页内容进行分析：
  - 判断是否包含用户所需信息
  - 提取关键信息点
  - 评估信息可信度
- 记录分析进度和状态

#### 2.2.5 结果汇总
- 综合20个网页的分析结果
- 统计信息命中情况
- 整理输出结构化结论
- 提供信息来源参考

## 3. 技术架构

### 3.1 技术栈
- **编程语言**：Python 3.10+
- **AI框架**：LangChain, LangGraph
- **大模型**：OLLAMA + qwen3.5:4b
- **网页爬取**：requests, BeautifulSoup4
- **搜索引擎**：百度搜索（API或网页抓取）

### 3.2 系统架构
```
用户输入
    ↓
[LangGraph Agent]
    ├── 需求理解节点
    ├── 搜索执行节点
    ├── 内容获取节点（循环处理20个网页）
    ├── AI分析节点（循环处理20个网页）
    └── 结果汇总节点
    ↓
输出结论
```

### 3.3 关键组件
1. **LangGraph工作流**：编排Agent的执行流程
2. **搜索引擎模块**：负责与百度搜索交互
3. **网页爬取模块**：获取和解析网页内容
4. **LLM集成模块**：通过LangChain调用OLLAMA模型
5. **结果管理模块**：存储和汇总分析结果

## 4. 非功能需求

### 4.1 性能要求
- 单次搜索与分析流程耗时：≤ 5分钟（20个网页）
- 支持并发处理多个网页

### 4.2 可靠性
- 处理网页访问失败的情况
- 记录错误日志
- 支持断点续传（可选）

### 4.3 可扩展性
- 支持更换不同的搜索引擎
- 支持更换不同的LLM模型
- 可调整分析的网页数量

## 5. 验收标准

1. 能够正确接收用户自然语言输入并执行搜索
2. 能够成功获取并分析至少20个搜索结果网页
3. 分析结果准确，能够识别包含目标信息的网页
4. 输出清晰、结构化的分析结论
5. 系统运行稳定，无重大错误

## 6. 开发计划

1. 环境搭建与依赖安装
2. 搜索引擎模块开发
3. 网页爬取模块开发
4. LangChain与OLLAMA集成
5. LangGraph工作流设计与实现
6. 结果汇总与输出功能开发
7. 集成测试与优化
